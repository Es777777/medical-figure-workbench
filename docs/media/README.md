# Media Assets

This directory holds presentation assets for the repository homepage and future project demos.

## Suggested Files

- `workbench-overview.png`
  - full product screenshot showing project toolbar, task list, canvas, and export panel
- `import-review-flow.gif`
  - short animation showing upload, split review, import, and export steps
- `resource-replacement.png`
  - screenshot of recommended resources and replacement workflow

## Capture Guidance

- Use a clean sample project with meaningful task names
- Show the project toolbar and task list when possible
- Prefer one screenshot for product structure and one for actual editing detail
- Keep image widths comfortable for GitHub README viewing
- Avoid sensitive manuscript content in demo captures

## Recommended Capture Order

1. Create a project with at least two figure tasks
2. Upload and split a manuscript figure
3. Review panel decisions
4. Import into canvas
5. Show export validation and export actions
6. Show the onboarding card before the first import for first-use documentation

## Notes

If media is not yet committed, the main `README.md` should still point here so contributors know exactly what assets to add next.
